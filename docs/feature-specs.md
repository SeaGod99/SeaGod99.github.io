# 水神的工具箱 — 功能規格規劃

> 為 FFXIV 繁體中文伺服器玩家打造的工具集合，記錄各功能應有的功能細節。
> 維護原則：優先做純計算、免維護功能；資料盡量走 API（XIVAPI v2 / Universalis / Teamcraft）；繁中名稱用簡中 datamining + OpenCC 轉換。

# 一、日常工具

## 1.1 天書奇談計算器 ✅
**路徑**：`/tools/wondrous-tails/`

核心功能
- 輸入目前已揭開的格子位置（4×4 共 16 格，貼上 9 個任務獎勵章）
- 以 Monte Carlo 模擬計算完成 1 / 2 / 3 連線的機率
- 「重貼」（Second Chance）建議：比較保留現況 vs 使用重貼點重抽的期望連線數
- 輸出建議：應保留還是重貼、各連線數的機率分布

應有但可補強
- 重貼點剩餘次數輸入，連動建議
- 視覺化 4×4 棋盤點擊輸入（取代純文字座標）
- 顯示每週獎勵清單（成長之痂/詩學等）對照

---

## 1.2 仙人微彩計算機 ✅
**路徑**：`/tools/cactpot/`

核心功能
- 輸入已翻開的格子數字（3×3 盤面）
- 期望值計算：建議下一張該翻哪格、最後該選哪條線（8 條線）
- 對照賠率表換算 MGP 期望收益

應有但可補強
- 「最小仙人微彩」（Mini Cactpot）與「仙人微彩」（Jumbo，每週開獎）切換
- MGP 期望值換算成常見兌換品（如笛子、地圖坐騎）需要的場次估算

---

## 1.3 限時採集節點查詢 🛠️
**路徑**：`/tools/gathering/`（已有頁面）

核心功能
- 列出限時採集節點（傳說/稀有物品）：物品名、採集職業（採礦/園藝）、地圖、座標
- 出現時間（Eorzea Time 起訖）與下次出現倒數計時
- 以現實時間換算 ET（ET = 現實時間 × 20.5714）即時更新倒數

應有功能
- 篩選：依職業、依版本、依地圖、依物品類型（素材/收藏品）
- 即將出現排序（最近要刷的排前面）
- 點擊節點顯示 Teamcraft 座標與 flag 指令（可複製 `/coord`）
- 我的追蹤清單：勾選關注節點，只看關注的倒數
- 收藏品節點標記（需高採集數值）

資料來源：Teamcraft nodes JSON（追版本更新）

---

## 1.4 B 怪討伐路線 📋
**規劃路徑**：`/tools/hunt-b/`

範圍限定：僅 B 級怪（不做 A/S，因 A/S 需即時聯動社群回報，維護成本高）

核心功能
- 各資料片地圖的 B 怪固定刷新點位置與座標
- 每日討伐路線建議：依地圖串連最短路徑
- 顯示 B 怪掉落（狩獵物資/獎章）與所需數量
- flag 座標可複製

應有功能
- 依資料片 / 地圖篩選
- 勾選已討伐，路線自動跳過
- B 怪重生時間提示（擊殺後冷卻）

---

## 1.5 風脈泉追蹤器 📋
**規劃路徑**：`/tools/aether-currents/`

核心功能
- 各飛行地圖的風脈泉清單：地圖採集型（10 個）+ 任務型風脈泉
- 標示每個風脈泉座標、採集型 flag、任務型來源 NPC/任務名
- 進度追蹤：勾選已收集，顯示「還差幾個可飛行」

應有功能
- 依地圖分組，顯示各地圖 x/總數 進度
- 進度本機儲存（localStorage / 匯出匯入 JSON）
- 採集型風脈泉地圖預覽圖（座標標點）

資料來源：A Realm Remapped / Teamcraft 風脈泉座標可參考

---

## 1.6 時尚品鑑推薦 📋
**規劃路徑**：`/tools/fashion-report/`

核心功能
- 顯示本週時尚品鑑（Fashion Report）主題與評分項目
- 各部位推薦裝備 / 染色組合（達 80 分以上）
- MGP 獎勵說明（首次達標 / 滿分差異）

維護注意：每週更新，需半自動（社群解答抓取或手動填）。建議做成讀取一份每週更新的 JSON。

# 二、收藏 / 成就追蹤

> 共通設計：所有收藏追蹤頁面採用同一套 UI 與資料結構，差異只在資料來源與分類。

## 收藏追蹤共通規格

每個收藏頁面都應具備：
- **總進度條**：已取得 / 總數，百分比
- **分類分頁**：依取得方式（任務/副本/商店/活動/成就/採集等）分組
- **每個項目卡片**：圖示、名稱（繁中）、取得方式說明、是否已取得勾選
- **篩選**：已取得 / 未取得 / 全部；關鍵字搜尋
- **進度保存**：localStorage 儲存，提供匯出 / 匯入 JSON（換裝置可帶走）
- **資料來源標註**：每個項目可連結灰機 Wiki / Teamcraft

維護注意：收藏品「取得方式」需手動對照 wiki，是最耗工部分，每大版本需補充。

# 三、戰鬥 / 副本

## 3.1 配裝規劃器 ✅（外部連結）

## 3.2 冒險者小隊計算機 📋
**規劃路徑**：`/tools/squadron/`

核心功能
- 輸入小隊成員屬性（體能/精神/戰術），計算是否能通過各訓練

應有功能
- 化學實驗（屬性加成）效果模擬

## 3.3 優雷卡 / 禁地天氣・NM 📋
**規劃路徑**：`/tools/eureka/`

- 優雷卡各區域（常風/不凍/泡影/隱秘）天氣即時顯示與預報（純演算法）
- NM（Notorious Monster）湧現條件：所需天氣 + 時段
- 下次符合條件 NM 的倒數計時
- 同步可做禁地兵裝（Bozja）天氣・CE/迷宮提示

資料來源：天氣純演算法計算（同天氣預報工具），NM 條件表手動整理

# 四、生活職（採集 / 製作 / 市場）

## 4.1 市場查價工具 ✅
**路徑**：`/market/`

核心功能
- 查詢物品在各伺服器 / DC 的即時上架價、近期成交價
- 製作成本分析（材料總價 vs 成品售價）

應有但可補強
- 跨 DC 比價（哪個服最便宜）
- 價格歷史走勢圖
- 物品收藏清單快速查價

資料來源：Universalis（繁中服 ID 4028–4035，Elemental DC）

## 4.2 製作利潤計算機 📋（繁中空缺，優先）
**規劃路徑**：`/tools/crafting-profit/`

核心功能
- 選擇要製作的成品 → 自動展開材料樹（多層配方）
- 每個材料查 Universalis 市價，計算：
  - 全買材料製作成本
  - 成品售價（HQ/NQ）
  - 預估利潤 / 利潤率
- 標示哪些材料「買成品比買料便宜」（建議直接買半成品）

應有功能
- 數量設定（做幾個），總成本/總利潤
- 採集材料標記（可採集的不計入成本，或單獨估算）
- 伺服器 / DC 切換
- 排序：依利潤率找「現在做什麼賺」

參考：Frozen Rabbit's Workshop（備料 + 比價），定位為互補/整合而非重複

## 4.3 物品 / 製作搜尋 📋
**規劃路徑**：`/tools/item-search/`

核心功能
- 關鍵字搜尋物品（繁中名稱）
- 顯示物品資訊：圖示、類別、可否上市場、來源（製作/採集/掉落/商店）
- 若可製作：顯示配方、職業、等級
- 連結到市場查價、製作利潤計算

資料來源：XIVAPI v2 物品/配方；繁中名稱用簡中 datamining + OpenCC

## 4.4 藏寶圖採集點查詢 📋
**規劃路徑**：`/tools/treasure-maps/`

範圍：G8～G18（含現行版本）

核心功能

# 五、跨功能共通建議

1. **進度保存統一**：所有「追蹤類」功能（收藏、風脈泉、釣魚、採集、幻卡等）共用一套 localStorage + 匯出/匯入 JSON 機制，最好能整合成一個帳號式的本機存檔。
2. **天氣演算法共用模組**：天氣預報、探索筆記、優雷卡、釣魚都要用到 ET/天氣計算，抽成共用 JS module（`assets/js/eorzea-time.js` 之類）。
3. **繁中名稱共用模組**：物品/技能繁中名稱統一走「簡中 datamining + OpenCC」的同一份資料與轉換邏輯，避免各頁面各自處理。
4. **flag 座標複製**：採集、藏寶圖、B 怪、風脈泉都需要「複製 `/coord` 指令」功能，抽成共用元件。
5. **版本標註**：資料來源於特定版本（採集點、配方）的頁面應標明對應遊戲版本，方便大版本更新時排查。

# 六、建議開發優先序

依「免維護程度 × 繁中空缺程度」排序：

| 優先 | 功能 | 理由 |
|------|------|------|
| 1 | 製作利潤計算機 | 繁中空缺、純計算（走 API）、需求高 |
| 2 | 限時採集節點查詢（補強） | 已有骨架、純演算法倒數 |
| 3 | 風脈泉追蹤器 | 純資料 + 進度追蹤，免持續維護 |
| 4 | 優雷卡天氣・NM | 天氣純演算法，可重用模組 |
| 5 | 物品/製作搜尋 | API 驅動，是製作利潤的基礎 |
| 6 | 探索筆記 / 釣魚 | 重用天氣模組 |
| 7 | 各收藏追蹤 | 量大且需手動維護來源資料，分批做 |
